This simple action moves an issue to a specified column in a GitHub V2 Project
(e.g., from `Todo` to `In Progress`).

## Usage

First, create a PAT or GitHub App installation token with the `repo` and
`projects` scopes. The default `GITHUB_TOKEN` is scoped to the repository level
and cannot access projects.

- [Create a new PAT](https://github.com/settings/tokens/new?scopes=repo,project&description=MY_PAT)
  with `repo` and `projects` scopes.
- [Add a secret](https://docs.github.com/en/actions/how-tos/write-workflows/choose-what-workflows-do/use-secrets)
  `MY_PAT` with the personal access token.
- Update the `github-token` in the workflow `.yml`:

```yaml
github-token: ${{ secrets.MY_PAT }}
```

### Create Project

- Create a new project in your repository and select `New Board`.
- Note down the project's number from the URL (e.g.,
  `https://github.com/users/<foo>/projects/3`).
- Make sure there's the `Status` field, e.g.
  `https://github.com/users/<foo>/projects/3/settings/fields/Status`.
- Go to your project -> Workflows -> Enable `Auto-add to project`

### Run Example Workflow

Here's an example of how to use this action in a workflow file:

```yaml
name: Example Workflow (manual)

on:
  workflow_dispatch:
    inputs:
      issue-number:
        description: The number of the issue you want to move around.
        required: true
        type: number
      project-number:
        description: The Project number which you can find in the project's URL.
        required: true
        type: number
      column-name:
        description: The project column name to move the issue to (e.g., Done).
        required: true
        type: string
      repo-name:
        description: Repository name (optional, defaults to current repo name).
        required: false
        type: string
      repo-owner:
        description:
          Repository owner (optional, defaults to current repo owner).
        required: false
        type: string

permissions: read-all

jobs:
  move-issue:
    runs-on: self-hosted
    steps:
      - name: Move issue to project column
        uses: ./
        with:
          github-token: ${{ secrets.MY_PAT }}
          issue-number: ${{ inputs.issue-number }}
          project-number: ${{ inputs.project-number }}
          column-name: ${{ inputs.column-name }}
          repo-name: ${{ inputs.repo-name }}
          repo-owner: ${{ inputs.repo-owner }}
```

## Inputs

| Input            | Description                                                                                                        |
| ---------------- | ------------------------------------------------------------------------------------------------------------------ |
| `github-token`   | Your PAT or GitHub App installation token.                                                                         |
| `issue-number`   | The number of the issue to move. The issue already needs to be part of the project.                                |
| `project-number` | The number of the project. You get in from the project URL (e.g., `https://github.com/users/stulle123/projects/2`) |
| `column-name`    | The project column name to move the issue to (e.g., `Done`).                                                       |
| `repo-name`      | Optional: The repository name.                                                                                     |
| `repo-owner`     | Optional: The repository owner.                                                                                    |

## Development

- Download and install Node.js 20.x, which includes npm
- `cd move-project-column-action` and `npm install` to install dependencies
- Make changes in `src/` as needed
- Format: `npm run format:write`
- Lint: `npm run lint`
- Run tests: `npm run test`
- Run coverage: `npm run coverage`
- Compile your `dist/index.js` package file: `npm run package`
- Commit and push changes
